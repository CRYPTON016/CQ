Project 3

Preparation:

  • Initialize Google Cloud Shell: This provides a command-line environment within your GCP console.
  • Enable Billing: Honeypot deployments on GCP will incur charges, so ensure billing is enabled for your project.
  • Enable Compute Engine API: The Compute Engine API allows you to create and manage virtual machines (VMs) needed for the honeypot.
  • Select Region/Zone: Choose a region and zone where you want to deploy your honeypot VM. This affects latency and costs.

Deployment:

  • Configure Firewall: Since a honeypot acts as a decoy, you'll need to configure firewall rules to allow specific traffic to reach the honeypot VM while blocking everything else.
  • Install MHN Admin: Many honeypot solutions use a central management tool called MHN Admin. You'll need to install it on a separate VM.
  • Honeypot VM Setup: Create a VM instance for the honeypot itself. Choose an appropriate operating system based on the honeypot software you select.
Additional Considerations: Security: Carefully manage firewall rules to avoid exposing unintended services on your honeypot VM. Monitoring: Set up monitoring for your honeypot VM to track its health and potential attacks. Legality: Operating a honeypot might have legal implications depending on your location. Ensure you comply with relevant laws.
